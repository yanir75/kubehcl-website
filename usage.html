Usage

KubeHCL lets you define Kubernetes resources using HCL instead of YAML. Then, run kubehcl apply to deploy them.

YAML Example


apiVersion: v1
kind: Namespace
metadata:
  name: foo
  labels:
    name: bar

Equivalent HCL (KubeHCL)


kube_resource "foo" {
  apiVersion = "v1"
  kind       = "Namespace"

  metadata = {
    name = "foo"
    labels = {
      name = "bar"
    }
  }
}
