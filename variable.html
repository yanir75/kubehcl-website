variable block

The variable block defines input variables that can be reused throughout your configuration. It contains three optional attributes:

  • description – explanation of the variable usage (optional).
  • type – constraint on the variable value, must match the type (optional).
  • default – value which will be assigned if not defined elsewhere (optional).

Variables can be referenced in configuration files using the syntax var.<name>, for example: var.foo.

Example


variable "replicas" {
  description = "Number of replicas for the deployment"
  type        = number
  default     = 3
}
