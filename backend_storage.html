backend_storage block

The backend_storage block defines where the state of your configuration is stored. Valid options are:

  • stateless – applies configuration to all resources in the files, even those not managed by kubehcl.
  • kube_secret – stores the state as a Kubernetes Secret (default option).

This block is optional. If not defined, kube_secret will be used as the default backend.

Example


backend_storage {
    kube_secret {}
}
